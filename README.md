# 🛡️ CyberSec AI — чат-бот этичного хакера

> **CyberSec AI** — учебный браузерный помощник по defensive security, Linux, сетям, SOC, CTF и легальному пентесту. Он работает **локально без AI API** и при желании подключается к защищённому Node.js backend.

<p align="center">
  <strong>💻 Терминальный UI</strong> · <strong>🧠 Local RAG</strong> · <strong>🔐 Defensive Security</strong> · <strong>⚡ Optional AI Backend</strong>
</p>

---

## ⚖️ Этическое назначение

CyberSec AI создан **только** для:

- ✅ обучения информационной безопасности;
- ✅ защиты собственных систем;
- ✅ лабораторных стендов и CTF;
- ✅ аудита с письменным разрешением владельца;
- ✅ анализа логов, конфигураций и инцидентов в защитном контексте.

Он **не предназначен** для взлома чужих аккаунтов, фишинга, DDoS, malware, обхода авторизации, закрепления в чужих системах или сокрытия следов.

## 🚀 Быстрый старт

### 1️⃣ Установите Node.js

Скачайте актуальную **LTS**-версию с <https://nodejs.org/> или установите через пакетный менеджер вашей ОС.

Проверьте установку:

```bash
node --version
npm --version
```

### 2️⃣ Установите зависимости

```bash
npm install
```

### 3️⃣ Создайте `.env`

```bash
cp .env.example .env
```

Для полностью локального режима оставьте ключ пустым:

```env
PORT=3000
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4.1-mini
CORS_ORIGIN=http://localhost:3000
```

### 4️⃣ Запустите сервер

```bash
npm start
```

### 5️⃣ Откройте приложение

Перейдите в браузере на:

```text
http://localhost:3000
```

## 🧠 Как работает локальный режим

Локальный режим не требует OpenAI API и работает прямо в браузере по JSON-файлам.

```text
Сообщение пользователя
        │
        ▼
🛡️ Safety check опасных запросов
        │
        ▼
❓ FAQ exact match
        │
        ├─ найдено → быстрый ответ FAQ
        │
        ▼
🧰 Анализ команды
        │
        ├─ команда найдена → разбор синтаксиса, риска и безопасного варианта
        │
        ▼
🧾 Анализ логов
        │
        ├─ похоже на логи → IP, порты, ошибки, гипотезы, план диагностики
        │
        ▼
✅ Чек-лист
        │
        ├─ запрос чек-листа → защитный checklist
        │
        ▼
🔎 Local RAG по knowledge.json
        │
        ├─ найден документ → ответ с источником и релевантностью
        │
        ▼
💬 Уточняющий fallback
```

## 🔎 Как устроен простой RAG

RAG здесь намеренно простой, прозрачный и удобный для обучения:

1. 🧹 запрос нормализуется в нижний регистр;
2. ✂️ текст разбивается на ключевые слова;
3. 📄 каждый документ из `knowledge.json` превращается в набор слов;
4. 📊 считается пересечение слов запроса с документом;
5. 🏆 выбирается документ с лучшей примерной релевантностью;
6. 🧾 ответ показывает источник локального знания.

> ⚠️ Ограничение: это не семантический поиск. Нет эмбеддингов, морфологии, reranking и внешней проверки фактов. Зато логику легко понять, изменить и расширить.

## 🤖 AI-режим

AI-режим включается переключателем в интерфейсе. Frontend отправляет на backend:

- сообщение пользователя;
- выбранный режим;
- короткий контекст последних сообщений.

Backend:

- 🔐 читает `OPENAI_API_KEY` только из переменных окружения;
- 🧱 добавляет системный defensive security prompt;
- 📏 ограничивает размер входного сообщения;
- 🧪 валидирует типы данных;
- 🧯 скрывает внутренние ошибки;
- 🚦 применяет базовый rate limit;
- 🔁 возвращает локальный fallback, если OpenAI API недоступен.

### Почему API-ключ нельзя хранить во frontend

Frontend-код загружается в браузер пользователя. Любой ключ внутри HTML/JS/CSS можно увидеть в DevTools, скачать из вкладки Network или извлечь из исходников. Поэтому ключ хранится только на backend, в `.env`, секрет-хранилище или переменных окружения платформы.

## 🛡️ Реализованные меры безопасности

- 🚫 безопасный отказ от вредоносных запросов;
- 🔐 API-ключ не передаётся во frontend;
- 🧼 пользовательский текст вставляется через `textContent`, без небезопасного `innerHTML`;
- 🕵️ сообщения, похожие на секреты, не сохраняются в `localStorage`;
- 🧱 backend не выполняет команды ОС;
- 🧨 нет `eval` и передачи пользовательского ввода в shell;
- 🧾 логи и документы пользователя считаются недоверенными данными;
- 📏 ограничены размер JSON и длина сообщения;
- 🚦 добавлен базовый rate limit;
- 🧯 внутренние ошибки сервера скрываются от клиента;
- 🧭 AI получает системный prompt с defensive-only рамками.

## 🧰 Как добавлять новые знания

### Добавить статью в `knowledge.json`

```json
{
  "id": "kb-new",
  "title": "Новая тема",
  "category": "защита серверов",
  "keywords": ["linux", "hardening", "server"],
  "content": "Краткая защитная статья с практическими рекомендациями."
}
```

### Добавить команду в `commands.json`

```json
{
  "name": "example-command",
  "os": "Linux",
  "category": "диагностика",
  "syntax": "example-command --safe-flag",
  "purpose": "Объясняет безопасное назначение команды.",
  "example": "example-command --safe-flag",
  "riskLevel": "низкий",
  "warning": "Не запускайте на чужих системах без разрешения.",
  "requiresRoot": false
}
```

## 🧪 Как протестировать fallback

| Сценарий | Ожидаемое поведение |
| --- | --- |
| `OPENAI_API_KEY` пустой, AI toggle включён | Backend вернёт `LOCAL_FALLBACK` |
| OpenAI API недоступен | Backend не раскрывает ошибку и отвечает локально |
| Сервер остановлен | Frontend показывает ошибку AI и использует локальный режим, если JSON доступен |
| Повреждён JSON | Интерфейс показывает понятную ошибку загрузки базы |
| Пустое сообщение | Бот просит ввести задачу, команду или логи |

## 🌍 Размещение на VPS

Рекомендуемый безопасный порядок:

1. 🧑‍💻 создайте отдельного пользователя без root-доступа для приложения;
2. 📦 установите Node.js LTS и Nginx;
3. 📁 разместите проект, например в `/opt/cybersec-ai`;
4. 🔐 создайте `.env` и задайте права `chmod 600 .env`;
5. 📥 установите зависимости через `npm ci --omit=dev`;
6. 🧭 запустите приложение через `systemd` или PM2;
7. 🔥 откройте на firewall только 80/443;
8. 🧾 включите логирование и ротацию логов;
9. 💾 настройте резервное копирование `.env` отдельно и безопасно.

## 🌐 Reverse proxy через Nginx

Минимальный пример:

```nginx
server {
  listen 443 ssl http2;
  server_name cybersec.example.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

Для production добавьте:

- 🔒 TLS через Let's Encrypt;
- 🧱 security headers;
- 📏 лимиты размера тела запроса;
- 🚦 rate limit на уровне Nginx;
- 📜 access/error logs;
- 🔥 firewall с минимально открытыми портами.

## 🧭 FAQ vs RAG vs AI

| Источник | Когда используется | Плюсы | Ограничения |
| --- | --- | --- | --- |
| ❓ FAQ | Точное совпадение популярного вопроса | Быстро, стабильно, предсказуемо | Работает только на заранее добавленных вопросах |
| 🔎 RAG | Похожий вопрос по теме из базы знаний | Показывает источник, легко расширяется | Не понимает глубокую семантику |
| 🤖 AI | Сложный анализ, конфигурации, логи, объяснения | Гибкие ответы и контекст | Нужны backend, ключ, сеть и safety-контроль |

- 🏁 **CTF-помощник** — подсказки без полного решения сразу.
- 🧯 **Параноик** — строгая оценка рисков и hardening.
