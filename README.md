# CyberSec AI — чат-бот этичного хакера

CyberSec AI — учебный браузерный чат-бот по defensive security, Linux, сетям, SOC, CTF и легальному пентесту. Он работает локально по JSON-базам и может опционально отправлять запросы на Node.js backend с OpenAI API.

> Проект предназначен только для обучения, защиты собственных систем, лабораторных стендов, CTF и тестирования с разрешения владельца.

## Дерево проекта

```text
.
├── .env.example
├── README.md
├── commands.json
├── faq.json
├── index.html
├── knowledge.json
├── package.json
├── script.js
├── server.js
└── styles.css
```

## Структура

- `index.html` — интерфейс чат-бота, панели режимов, быстрые кнопки, форма сообщения.
- `styles.css` — адаптивный терминальный cybersecurity dashboard с тёмной/светлой темой.
- `script.js` — frontend-логика: локальный FAQ/RAG, анализ команд, логов, история, экспорт, fallback.
- `faq.json` — частые вопросы с приоритетом точного совпадения.
- `knowledge.json` — локальные документы для простой RAG-системы.
- `commands.json` — безопасный справочник диагностических команд.
- `server.js` — Express backend с `POST /api/chat`, rate limit, валидацией и AI fallback.
- `.env.example` — пример переменных окружения.
- `package.json` — зависимости и команды запуска.

## Установка Node.js

Установите актуальную LTS-версию Node.js с <https://nodejs.org/> или через менеджер пакетов вашей ОС. Проверьте установку:

```bash
node --version
npm --version
```

## Установка зависимостей

```bash
npm install
```

## Настройка `.env`

Скопируйте пример:

```bash
cp .env.example .env
```

Для полностью локального режима оставьте `OPENAI_API_KEY` пустым. Для AI-режима добавьте ключ только в `.env` на сервере:

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4.1-mini
PORT=3000
```

Никогда не храните API-ключ во frontend: JavaScript, HTML и CSS загружаются в браузер и доступны пользователю через DevTools.

## Запуск сервера

```bash
npm start
```

Откройте <http://localhost:3000>. Статические файлы раздаются Express, а AI-запросы идут на `/api/chat`.

## Локальный режим

Если AI backend выключен в интерфейсе или `OPENAI_API_KEY` отсутствует, бот отвечает локально:

1. Ищет точное совпадение в `faq.json`.
2. Пробует распознать команду и объяснить её через `commands.json`.
3. Пробует распознать фрагмент логов и выделить ошибки, IP, порты и гипотезы.
4. Для обычного вопроса выполняет простой RAG по `knowledge.json`.
5. Если совпадений нет, просит уточнить контекст.

## AI-режим

При включённом переключателе frontend отправляет на backend только сообщение, режим и короткий контекст диалога. Backend добавляет системный промт defensive security, валидирует вход, ограничивает размер сообщения, применяет rate limit и скрывает внутренние ошибки.

Если OpenAI API недоступен, backend возвращает локальный fallback из JSON-базы.

## Как устроен простой RAG

RAG в этом проекте намеренно простой и понятный новичкам:

- запрос нормализуется в нижний регистр;
- текст разбивается на ключевые слова;
- для каждого документа считаются пересечения слов запроса с заголовком, категорией, ключевыми словами и содержимым;
- выбирается документ с лучшей примерной релевантностью;
- ответ показывает источник и процент совпадения.

Ограничения: нет морфологии, семантических эмбеддингов, ранжирования по смыслу и проверки фактов вне локальной базы.

## Как добавлять статьи и команды

Добавьте объект в `knowledge.json`:

```json
{
  "id": "kb-new",
  "title": "Новая тема",
  "category": "защита серверов",
  "keywords": ["linux", "hardening"],
  "content": "Краткая защитная статья..."
}
```

Добавьте объект в `commands.json` с полями `name`, `os`, `category`, `syntax`, `purpose`, `example`, `riskLevel`, `warning`, `requiresRoot`.

## Безопасность

Реализовано:

- отказ от вредоносных запросов во frontend и системном промте backend;
- API-ключ читается только из переменных окружения сервера;
- пользовательский текст вставляется через `textContent`, без небезопасного `innerHTML`;
- сообщения с похожими на секреты словами не сохраняются в `localStorage`;
- backend не выполняет команды ОС, не использует `eval` и не передаёт ввод в shell;
- пользовательские логи и документы считаются недоверенными данными;
- ограничение размера JSON и сообщения;
- базовый rate limit;
- скрытие внутренних ошибок.

## Размещение на VPS

1. Установите Node.js LTS и Nginx.
2. Склонируйте проект в `/opt/cybersec-ai`.
3. Создайте `.env` с секретами и правами `chmod 600 .env`.
4. Установите зависимости: `npm ci --omit=dev`.
5. Запустите через systemd или PM2 от отдельного пользователя без root.
6. Откройте только 80/443 на firewall, backend слушает localhost или закрытый порт.

## Reverse proxy Nginx

Минимальный пример:

```nginx
server {
  listen 443 ssl http2;
  server_name cybersec.example.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

Добавьте TLS через Let's Encrypt, security headers, лимиты размера тела и логирование.

## Архитектура приложения

Frontend загружает локальные JSON-файлы, хранит настройки и историю в браузере, выполняет локальный поиск и отображает ответы. Backend является опциональным доверенным слоем для AI API: он хранит ключ, добавляет системный промт, валидирует запрос и возвращает ответ или fallback.

## Порядок обработки сообщения

1. Пользователь вводит сообщение.
2. Frontend проверяет пустой ввод и опасные формулировки.
3. Если AI включён, запрос отправляется на `/api/chat`.
4. При успехе показывается источник `AI`.
5. При ошибке backend или выключенном AI используется локальная цепочка: FAQ → команды → логи → чек-лист → RAG → fallback.
6. Ответ печатается с терминальным эффектом и получает кнопки оценки, копирования и повторной генерации.

## FAQ vs RAG vs AI

- FAQ — лучший вариант для точных популярных вопросов: быстрый и предсказуемый.
- RAG — поиск по локальной базе знаний, полезен для похожих формулировок.
- AI — гибкий ответ на сложные вопросы, анализ конфигураций и логов, но требует backend, ключ и сетевой доступ.

## Как протестировать fallback

- Оставьте `OPENAI_API_KEY` пустым и включите AI в интерфейсе: backend вернёт `LOCAL_FALLBACK`.
- Остановите сервер и откройте `index.html` как статический файл: frontend покажет ошибку AI и продолжит локально, если JSON доступен.
- Временно испортите один JSON-файл, чтобы увидеть обработку ошибки загрузки.
